Название проекта: «Ретро-питончик». Классическая «Змейка»: игрок управляет змейкой на клеточном поле, ест еду, растёт и избегает столкновений со стенами и собственным телом. Дополнительно – бонусы, скины и режимы.
Игровая идея и цели:
Набрать как можно больше очков, поедая еду. Каждый съеденный объект увеличивает длину змейки на 1 сегмент и ускоряет игру (опционально). Столкновение с границей (или стенами уровня) или собственным хвостом – поражение. В некоторых режимах есть победные условия (например, «достичь длины 30»).

- [ ] задача не выполненa
- [x] задача выполненa

**Обязательные функции и механики (перечень):**

- [x] 1.	Клеточное поле (например, 30×20 клеток, размер клетки 24 px; окно 720×480) 
- [x] 3.	Змейка как список сегментов на сетке; голова задаёт направление движения
- [x] 4.	Движение дискретными шагами по таймеру (тик, например, каждые 120–200 мс; шаг ускоряется)
- [x] 5.	Еда появляется на свободной клетке, не совпадает со змеёй/стенами
- [x] 6.	Рост: при поедании – увеличение длины, инкремент очков
- [x] 7.	Столкновения: с границами (если режим без «порталов»), со стенами уровня, с собственным телом
- [x] 8.	Пауза/продолжение, перезапуск уровня
- [x] 9.	Табло очков, текущая длина, лучший результат (сохраняется *в json-файл*)
   
**Бонусы (перечень, минимум два вида):**

- [x] 1.	«+2» – добавляет два сегмента вместо одного
- [x] 2.	«Замедление» – временно увеличивает интервал тика
- [ ] 3.	«Призрак» (опционально) – на 3–5 секунд игнор столкновений с телом
- [ ] 4.	«Магнит» (опционально) – притягивает еду в радиусе R на один шаг
          Бонусы появляются реже обычной еды и имеют таймер исчезновения.

**Режимы игры (минимум два):**

- [x] 1.	Классический – без телепортов; ускорение с ростом
- [x] 2.	С портальными краями – выход за левый край переносит на правый и т.д.
- [ ] 3.	Хардкор (опционально) – фиксированная высокая скорость, стены на поле
- [ ] 4.	Испытание на длину (опционально) – цель: достичь заданной длины за ограниченное время

**Скины (перечень):**

- [x] 1.	Базовый – однотонные сегменты
- [ ] 2.	«Неон» – сегменты с контуром
- [ ] 3.	«Пиксель» – текстурные спрайты головы/хвоста

Скины переключаются в настройках; хранить выбранный скин в сохранениях.

**Управление и интерфейс (перечень):**

- [x] 1.	Стрелки или WASD – направление (запрет мгновенного разворота на 180°)(у меня просто стрелки)
- [x] 2.	P или Esc – пауза/меню паузы
- [x] 3.	R – перезапуск текущей попытки
- [x] 4.	В HUD: очки, длина, скорость/тик, активный бонус и время его действия
- [x] 5.	Звуки: поедание, бонус, столкновение, кнопки меню

**Структура уровней/карт (перечень):**

- [x] 1.	Не менее 5 карт с разным расположением стен/препятствий  (у меня генерятся рандомно)
- [x] 2.	Формат JSON: размеры сетки, список стен (координаты), стартовые координаты змейки и направление, настройки режима (порталы, базовая скорость, ускорение, наличие бонусов)
- [x] 3.	Генерация еды/бонусов не допускает спавн внутри стены или змейки
